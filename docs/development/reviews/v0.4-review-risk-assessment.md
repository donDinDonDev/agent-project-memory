# v0.4 Review And Risk Assessment

Date: 2026-06-07

## Verdict

The v0.4 API surface release track was reviewed and could proceed toward release prep.
No release-blocking findings remained.

This summary records the public review outcome only.

## Findings

No release-blocking findings were identified.

Future work already recorded by the v0.4 evaluation remained non-blocking for this
release:

- Reduce duplicated legacy generator warning presentation when both generated-source
  and hidden HTTP plugin warnings describe the same Maven code-generation plugin.
- Consider future `inputSpec` path discovery for generator plugin declarations that do
  not use supported spec filenames.
- Consider a future explicit generated-source scan mode, separate from the default
  deterministic source-visible analyzer.

## Review Coverage

Parser safety and malformed spec handling:

- OpenAPI/Swagger parsing uses bounded input size, stream read constraints, strict
  duplicate detection, and safe YAML loading limits.
- Unsupported or malformed spec shapes are caught and serialized as hidden warnings, not
  propagated as scan crashes.
- Duplicate normalized operation keys are collapsed into one warning signal instead of
  colliding operation facts.
- `operationId` and `tags` are bounded before serialization and evidence excerpting.
- The v0.4 implementation does not fetch network resources and does not resolve
  external `$ref` targets.

Path, generated-source, and evidence safety:

- Spec discovery uses repository containment checks and excludes ignored/generated
  paths from spec facts.
- Generated-source paths are warning-only `path_signal` evidence. Generated source
  contents are not read for endpoint/API facts in default scan mode.
- Evidence paths reviewed in docs, tests, goldens, and evaluation remain
  repository-relative, slash-separated, not absolute, not `./` prefixed, and not
  escaping via `..`.
- v0.4 spec and generated-source path IDs percent-encode unsafe path characters.

Category separation:

- Source-visible Spring MVC endpoints remain in `endpoints[]`.
- Interface-declared endpoints remain separate from direct source-visible endpoint
  facts.
- OpenAPI/Swagger operations are emitted under `api_surface.openapi.operations`, not
  `endpoints[]`.
- Generated-source API signals, repository-rest warnings, and hidden HTTP warnings
  remain warning/reference sections and do not create endpoint or operation facts.
