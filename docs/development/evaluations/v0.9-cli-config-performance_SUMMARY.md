# v0.9 CLI, Config, And Performance Evaluation Summary

Evaluation date: 2026-06-11

This public summary records the v0.9 validation for packaged CLI behavior, safe scan
configuration, deterministic output generation, and bounded local performance. It is a
validation summary, not a release note and not a performance guarantee.

## Scope

The evaluation covered representative packaged CLI behavior and local fixture copies:

- default Spring MVC scan behavior;
- multi-module Maven/build-config scan behavior;
- quality-guide regression fixture behavior;
- root-local scan config include/exclude behavior for local Markdown discovery;
- explicit config selection behavior;
- `local_markdown: false`;
- invalid config failure behavior;
- help, version, and usage-error CLI behavior;
- deterministic output stability;
- a medium tracked-source archive scan for performance sanity.

The evaluation did not change analyzer behavior, generated output contracts, evidence
semantics, packaging behavior, release versioning, publication state, or installation
channels.

## Results

- Local packaging and packaged CLI validation passed for the evaluated build.
- Packaged `--help`, `help`, `scan --help`, `--version`, and `version` commands
  succeeded without scanning and kept stderr empty.
- Usage errors and invalid config failures returned the documented exit-code categories.
- Default scans over representative fixture copies completed successfully and emitted
  `schema_version: "0.9"` with redacted `scan` metadata.
- Root-local config include/exclude rules affected only local Markdown discovery:
  explicitly included Markdown was accepted, excluded Markdown was omitted, and Java/
  Spring scan output stayed unaffected.
- Explicit config selection recorded the normalized repository-relative config path and
  did not create tool-config evidence records.
- `local_markdown: false` disabled local document discovery, reconciliation, guide
  rendering from local documents, and document evidence without changing Java/Spring
  scan output.
- Invalid config for a reserved scan mode did not write partial project-map output.
- Deterministic output checks passed for default, config, disabled-local-Markdown,
  explicit-config, and tracked-source archive targets.

## Performance Sanity

The short packaged-CLI timing pass stayed consistent with the earlier v0.9 performance
baseline. Small fixture scans completed in roughly half a second, where JVM startup and
CLI process overhead dominate. The medium tracked-source archive completed in under five
seconds across measured runs.

These numbers are local sanity observations only. They are not release promises and
should be rechecked after meaningful parser, packaging, filesystem, or analyzer changes.

## Observations

- No CLI automation, config, output-determinism, or performance finding requiring an
  implementation fix was observed in this evaluation.
- The evaluated development artifact still reported the previous release version before
  v0.9 release preparation; generated outputs correctly used `schema_version: "0.9"`.
- The earlier deferred performance follow-up remains valid: a shared source parse model
  could reduce repeated analyzer parsing work, but it needs a separate design and review
  boundary because it would touch analyzer internals.
- Release readiness for v0.9 should continue to verify packaged help/version behavior,
  packaged scan behavior, checksum assets, and public release documentation after the
  release version is set.
