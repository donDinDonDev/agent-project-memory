## Observations

- Spring Data repository interfaces were emitted as inferred repository signals with
  `entity_relation_status: "not_analyzed"`.
- Configuration classes, configuration-properties types, and bean methods were emitted
  without runtime bean graph, binding, or bean-name claims.
- Transaction, scheduled, event-listener, and messaging listener observations remained
  source-visible annotation signals without runtime behavior or topology claims.
- Messaging listener outputs did not serialize destination-like annotation values.
- Spring Security configuration output stayed warning-only and did not claim policy,
  endpoint protection, authentication, authorization, vulnerabilities, or correctness.

## Limitations And Follow-Ups

- A future hardening pass may evaluate symbol-only annotation evidence excerpts if a
  stricter evidence excerpt policy is desired.
- Direct `@EventListener` and direct `@RabbitListener` real-project coverage can be
  expanded with future evaluation targets or focused fixtures.
- Source-root compatibility for non-Maven Spring Boot guide inputs may need clearer
  public documentation if it remains supported.
- Runtime bean graph reconstruction, autowiring, conditional evaluation, repository
  query semantics, transaction runtime behavior, scheduler registration, messaging
  topology, broker configuration, and Spring Security policy remained out of scope.
