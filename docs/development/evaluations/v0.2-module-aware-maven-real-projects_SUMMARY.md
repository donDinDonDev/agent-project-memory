## Observations

- Root-declared Maven child modules were represented as module inventory items with
  repository-relative module IDs, support status, source/test roots, declaration
  evidence, and child POM evidence.
- Module-owned endpoint, component, entity, and test facts carried direct `module_id`
  fields.
- Unsupported child modules were reported as `maven_module` warnings and were not
  expanded into application facts.
- Zero-endpoint output for a supported module with no Spring MVC endpoint annotations
  stayed contract-aligned.

## Limitations

- Maven profile resolution, effective POM reconstruction, dependency graph
  reconstruction, Maven execution, generated sources, OpenAPI parsing, runtime Spring
  mappings, and test execution remained out of scope.
- The evaluation used two bounded public targets, not a broad corpus.
