# v3.4 Evidence Handoff and Repeatability Evaluation Summary

This summary records a narrow evidence-handoff and repeatability evaluation for a
local-first Java/Spring CLI/devtool that scans local inputs and writes deterministic
`.project-memory/` artifacts with source-backed evidence. This is existing product
baseline only: it is not a new product capability claim, release outcome, public proof,
SaaS/repository-chat/generic-RAG claim, provider-AI authority claim, source-upload
claim, automatic-code-modification claim, or broader language/runtime support claim.

The evaluation covered five observation areas: an evidence-first artifact reading path,
first-run orientation on `examples/quickstart-demo`, handoff navigation on
`examples/stage3-project-map`, selected query stdout repeatability on one
`spring-petclinic` artifact set, and selected heavy optional profile repeatability on
`high-performance-java-persistence`. These observations are limited to the samples,
projects, command shapes, and profile selector named below; they do not provide
benchmark proof, product proof, adoption or onboarding-success evidence, productivity
or token/time/context-savings evidence, security or privacy proof, performance or
resource proof, runtime-correctness proof, release-safety proof, universal determinism,
market-readiness evidence, or product-effectiveness evidence.

## Evidence-First Reading Path

A safe reading path starts with `artifact-set.json` for generated-set inventory and
authority labels, uses generated Markdown, query stdout, `agent-context`, `impact`, and
profile surfaces only for orientation, and resolves important claims through
`evidence-index.jsonl` plus cited source readback. This reading-path observation is
bounded to one fake quickstart sample and one fake checked-in stage3 snapshot;
`artifact-set.json` is contract/provenance metadata, not project evidence, and generated
Markdown, query stdout, retained packets, scorer notes, downstream-agent output,
profile output, and LLM/AI output remain non-evidence.

Query evidence lookup can present an evidence record and point readers to exact source
locators, but query stdout is deterministic presentation, not source-file readback. This
boundary applies to the current read-only no-adapter query surface; important decisions
still need `evidence-index.jsonl` and the cited source location.

## What This Summary Cannot Support

This summary does not support benchmark proof or product-proof outcomes. The evaluated
material is limited to the selected observation areas above and does not include
benchmark row outcomes, comparisons, final answers, or independent evidence of artifact
usefulness.

This summary does not support adoption, onboarding success, productivity, token savings,
time savings, context savings, downstream-agent superiority, or product effectiveness.
The quickstart and handoff observations use fake/local or checked-in sample material,
and scorer notes, downstream-agent output, local observations, and LLM/AI output are not
evidence for real-user outcomes.

This summary does not support universal determinism or broad repeatability. The
repeatability observations are limited to the named projects, selected command shapes,
selected impact input, and selected profile selector, and their matching digests or
diffs are output-repeatability observations only.

This summary does not support performance/resource proof, runtime correctness, release
safety, security or privacy proof, vulnerability-scanner claims, complete secret
detection, certification, compliance, market readiness, or public release readiness. No
CPU, memory, RSS, runtime, release, checksum, signing, SBOM, package-channel, security
scan, privacy review, or compliance evidence is included in this summary.

This summary does not change the product boundary. It does not introduce SaaS, hosted
scanning, repository chat, generic RAG, provider AI authority, source upload by default,
live connector truth, remote mutation, or automatic code modification.

## Evidence and Trust Boundary

Use `evidence-index.jsonl` and cited source readback to confirm important facts.
`project-map.json` contains structured generated facts that should resolve to evidence
for important use. `artifact-set.json`, generated Markdown, query stdout,
`agent-context`, `impact`, profile output, cache/workspace/adapter/AI presentation
surfaces, scorer notes, downstream-agent output, chat output, and LLM/AI output are
navigation or presentation only; missing evidence cannot be repaired by those surfaces.
