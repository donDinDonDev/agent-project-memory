# v0.6 JPA Domain Corpus Expansion Evaluation Summary

Evaluation date: 2026-06-09

## Scope

This public summary records optional expanded corpus coverage for the published v0.6
JPA/domain analyzer. The evaluation added public Maven project coverage for
source-visible JPA signals that were not exercised by the original v0.6 real-project
corpus: `@Embeddable`, `@EmbeddedId`, `@IdClass`, `@Enumerated`, and `@Version`.

This was validation-only work. It did not change analyzer behavior, generated output
schemas, evidence semantics, fixtures, goldens, tests, or published release assets.

## Observations

- The optional corpus expansion exercised every target signal in generated v0.6 output
  through pinned public Maven repositories.
- Embedded target linking stayed conservative. `@Embedded` and `@EmbeddedId` fields
  linked only to unique source-visible local `@Embeddable` facts; no runtime ORM
  resolution, provider metadata, database schema, or composite-key semantics were
  inferred.
- `@IdClass` output preserved source-visible type literals and kept field matching and
  semantic reconstruction as `not_analyzed`.
- Relationship entries preserved `target_resolution: "declared_type_only"` and
  `uncertainty: "target_type_not_resolved"` rather than claiming resolved entity
  targets.
- Missing annotation attributes remained absent or `null`; the generated output did not
  fill JPA provider defaults.
- The High-Performance Java Persistence target contains additional JPA examples under
  `src/test/java`, but the JPA/domain analyzer correctly limited entity facts to
  supported production source roots.

## Findings Triage

No release-blocking or follow-up-blocking finding was found. The expanded corpus did not
discover a need for a v0.6 implementation or contract correction.

## Limitations

- This evaluation improves real-project corpus coverage for v0.6 JPA/domain signals but
  is still not an exhaustive JPA corpus.
- Some selected targets are Maven JPA sample repositories rather than Spring Boot
  applications. They are useful for deterministic JPA annotation coverage but do not
  replace future Spring-heavy domain corpus expansion.
- Repository/entity inferred relations were not materially exercised by this follow-up;
  they were already covered by the original v0.6 evaluation and focused fixtures.
