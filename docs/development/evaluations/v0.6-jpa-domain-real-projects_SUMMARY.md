## Observations

- Spring PetClinic exercised the strongest v0.6 path: direct entity facts, mapped
  superclass identifiers, direct `@Column` field metadata, source-visible
  `@GeneratedValue` identifier metadata, relationship cardinality, join columns, one
  join table, and inferred Spring Data repository/entity generic relations.
- Relationship output stayed conservative. Relationship targets remained
  `target_resolution: "declared_type_only"` with `uncertainty:
  "target_type_not_resolved"` rather than claims about resolved entity targets, foreign
  keys, join-table existence, ORM ownership correctness, fetch behavior, cascade
  behavior, database constraints, provider defaults, or runtime ORM behavior.
- Repository/entity relations were emitted only when a supported Spring Data generic
  type matched exactly one emitted entity fact. Missing entity facts degraded to
  `not_detected` instead of unsafe links.
- Spring PetClinic REST showed that the initial conservative annotation-origin rules
  skipped common `import jakarta.persistence.*` entity classes, reducing real-project
  JPA and repository/entity relation coverage while preserving safety.
- Spring PetClinic Microservices used wildcard JPA imports across the evaluated modules.
  The v0.6 contract skipped those annotations, so no entity facts were emitted and
  Spring Data repository interface signals correctly stayed at `entity_relation_status:
  "not_detected"`.
- Spring Cloud OpenFeign worked as a no/low-JPA control: no entity or repository/entity
  relation facts were invented. The guide was safe but mildly noisy on this target.
- The selected real-project corpus did not exercise direct `@Embeddable`, `@Embedded`,
  `@EmbeddedId`, `@IdClass`, `@Enumerated`, or `@Version` output; those slices were
  covered by focused fixtures and later optional corpus expansion.

| No-domain guide rendering | Low | Guide rendering, tests/goldens, risk-based implementation review | Quiet no-domain `agent-guide.md` rendering when no entities, embeddables, domain relationships, or repository/entity relations are emitted. |
| Expanded corpus coverage | Low | Evaluation corpus only | Add real-project or focused real-world coverage for `@Embeddable`, `@EmbeddedId`, `@IdClass`, `@Enumerated`, and `@Version`. |

## Limitations

- Real-project coverage remained incomplete for embeddables, embedded IDs, id-class
  signals, enumerated fields, and version fields.
- Wildcard JPA import support was intentionally conservative in this evaluation, but it
  made real-project JPA coverage sparse on some targets.
- Guide rendering was safe but mildly noisy on no-domain targets.
