## Observations

- Spring PetClinic exercised the best current v0.6 path: direct entity facts, mapped
  superclass identifiers, direct `@Column` field metadata, source-visible
  `@GeneratedValue` identifier metadata, relationship cardinality, join columns, one
  join table, and three inferred Spring Data repository/entity generic relations.
- Relationship output stayed conservative. Every relationship target remained
  `target_resolution: "declared_type_only"` with `uncertainty:
  "target_type_not_resolved"`. The output did not claim resolved entity targets,
  foreign keys, join-table existence, ORM ownership correctness, fetch behavior, cascade
  behavior, database constraints, provider defaults, or runtime ORM behavior.
- Repository/entity relations stayed bounded. Relation objects were emitted only when a
  supported Spring Data generic type matched exactly one emitted entity fact. Missing
  entity facts degraded to `not_detected` instead of unsafe links.
- Spring PetClinic REST demonstrated a partial real-project coverage gap: classes with
  explicit single-type JPA imports were emitted, while common `import
  jakarta.persistence.*` entity classes were skipped under the current annotation-origin
  trust rules. This kept the analyzer safe but reduced entity and repository/entity
  relation coverage.
- Spring PetClinic Microservices contains JPA model classes, but those classes use
  wildcard JPA imports across the evaluated modules. The current v0.6 contract skips
  wildcard-only JPA annotations, so no entity facts were emitted and four Spring Data
  repository interface signals correctly stayed at `entity_relation_status:
  "not_detected"`.
- Spring Cloud OpenFeign worked as a no/low-JPA control for project-map safety: no
  entity or repository/entity relation facts were invented. However, `agent-guide.md`
  still rendered a `Detected JPA Entities` section and a persistence inspection hint
  with no entity evidence paths, which is harmless but noisy.
- None of the selected real projects exercised direct `@Embeddable`, `@Embedded`,
  `@EmbeddedId`, `@IdClass`, `@Enumerated`, or `@Version` output. Those slices remain
  covered by focused fixtures rather than this real-project corpus.
