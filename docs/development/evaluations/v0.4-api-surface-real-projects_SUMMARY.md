## Observations

- Supported local OpenAPI specs were discovered as declared API inputs.
- Minimal OpenAPI operations were emitted as `openapi_declared_operation` facts with
  `implementation_status: "not_analyzed"`, not as code-backed endpoint facts.
- Source endpoint evidence stayed separate from spec operation evidence.
- Generated-source and generator-plugin signals remained warnings.
- Generated-source path warnings were absent when generated roots were absent, preserving
  path-presence semantics.

## Limitations And Follow-Ups

- Duplicate legacy generator warning presentation is safe but mildly redundant.
- Endpoint/spec relation modeling remains future work and must preserve separate code
  evidence, spec evidence, confidence, and uncertainty.
- Explicit generated-source scan mode remains future work, must be non-default, and
  requires separate output/evidence contract design.
- Future plugin `inputSpec` discovery may be considered for specs that do not use common
  `openapi.*` or `swagger.*` filenames.
